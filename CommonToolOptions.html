Common Tool Options

I. Interface

Process ID Variable: The variable in the data stream which indicates the process in which the GUI element being acted upon is found. This should be a Process ID variable that has been output by an upstream tool.

Window Handle Variable: The variable in the data stream which indicates the window in which the GUI element being acted upon is found. This should be a window handle variable that has been output by an upstream tool. If this is not provided, the GUI element will be identified in the first window found by the tool.

Automation ID: The Automation ID of the GUI element being acted upon. If available, this is usually the best way to identify an element.

Name Regex: A regex string that will be run on the Name (Found using UIA Verify or some other automation api accessor tool) properties of the elements in the window to identify the element being acted upon. This can be the full name of the element. If a general keyword search is desired, you can surround your keyword with ".*" (ex: ".*TextInput.*").

Control Type (shown only when Click Location Type = "Element"): The control type property of the element being acted upon. For example, there may be a button with the name "select" as well as a dropdown with the name "select". In that scenario, setting the Name Regex field to "select" and the Control Type field to "Button" would ensure that the select button is identified instead of the select drop down

Index: If multiple elements are found, this specifies which one to return. For example, if you use a name regex selector of .*star.* when automating Microsoft Paint, there may be more than one GUI element that matches that regex string. You can then specify that you want the 0th element found (which is equivalent to the 1st) by specifying an index of 0. This should be used as a last resort when there is no other way to select an element.
